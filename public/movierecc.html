Project Overview

Built a real-time group recommendation system that helps multiple users agree on a movie to watch based on their individual ratings. Powered by a custom PyTorch model trained on the MovieLens dataset and served using FastAPI, this project simulates a collaborative filtering environment across a full-stack interface.

GitHub Repository: View Project on GitHub

Core Features

  • Room-based system – Users join a shared room to participate in group voting.
  • Rating & Submission – Each user rates 5 movies, and once all users submit, the model runs a prediction pass.
  • Bias correction – Implements residual-based bias correction so each user’s taste is fairly represented.
  • Top 3 recommendations – The group receives three optimized movie recommendations based on their combined inputs.

Technologies Used

  • Frontend: HTML, CSS, JavaScript (Vanilla)
  • Backend: FastAPI (Python), WebSockets
  • Machine Learning: PyTorch, trained on MovieLens 100k
  • Data: MovieLens dataset + genre vectors

Model Architecture

  • User and item embeddings
  • Genre vector fed through a dense layer
  • BatchNorm + Dropout for regularization
  • Bias correction applied per user based on prior residuals

REST + WebSocket APIs

  • WebSocket /ws – Handles real-time room communication
  • POST /api/recommend – Triggers the recommendation pipeline for a group
  • GET /api/movies – Returns movies to rate (optionally by genre)

User Interface

  • Minimal, responsive design with live updates via WebSocket
  • Genre dropdown, dynamic movie list with rating inputs
  • Displays final group recommendation as soon as all users finish
