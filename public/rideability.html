Project Overview

Developed a web-based application that provides a rideability score based on real-time weather conditions, allowing motorcyclists to make informed decisions about riding.

Core Features

  • Real-time Weather Data - Live weather info displayed based on an input ZIP code.
  • Rideability Score - Calculated using a predefined formula, with special weights for factors such as precipitation, visibility, and wind speed.

Technologies Used

  • Frontend: HTML, CSS, JavaScript
  • Backend: Node.js, Express.js
  • Database: PouchDB for storing and managing searches
  • API: Utilized OpenWeather API to fetch real-time weather data.

RESTful API

  • GET /api/weather/:zipCode: Fetches weather data and calculates the rideability score.
  • POST /api/searches: Creates a new search entry with weather data and rideability score.
  • PUT /api/searches/:id: Updates a search entry based on the unique ID.
  • DELETE /api/searches/:id: Deletes a specific search entry.

CRUD Functionality

Designed and implemented full **CRUD** (Create, Read, Update, Delete) endpoints for managing searches. Integrated PouchDB for seamless storage and retrieval.

User Interface

  • Responsive design using HTML, CSS, and JavaScript.
  • Features input validation, error handling for ZIP code inputs, and dynamic displays of weather data and rideability scores.
  • Showcases recent searches with delete capabilities.
