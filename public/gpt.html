Comparing GPT and Rosette for Program Synthesis

Team: Ern Kim, Jineshwar Nariani, Lana Vu, Manovay Sharma, Mishti Gala, Nikhila Peravali

Mentor: Angela Upreti

Project Overview

This project, conducted under the UMass URV program, explores the capabilities of GPT and Rosette in automating program synthesis. Our research focuses on tasks such as bug detection, synthesis, optimization, and verification, comparing the performance of both tools across diverse programming challenges.

Key Findings

  • Correctness: Rosette guarantees correctness if the specification is correct, while GPT may generate incorrect results, especially with vague specifications.
  • Specification: GPT’s specifications are easier to write but often lack precision. Rosette requires complete and formal specifications.
  • Program Space: Rosette operates on a small, safe subset of datatypes, whereas GPT can generate code in various languages, though its outputs are often incorrect.
  • Speed: GPT is faster in synthesis and bug detection, while Rosette excels in verification tasks.

Methods

We assessed three program synthesis tools: GenProg, Sketch, and Rosette, ultimately selecting Rosette due to its versatility, superior documentation, and ease of use. The research involved using GPT and Rosette to solve 20 diverse code generation problems, including the identification of problematic inputs and the synthesis of specific program sections.

Technologies and Tools Used

  • Tools: GPT, Rosette, GenProg, Sketch
  • Programming Language: Racket
  • Synthesis Approach: GPT leverages NLP algorithms for prediction, while Rosette uses SMT solver-aided search.

Future Work

  • Scaling up our initial findings to a larger set of synthesis tasks.
  • Exploring automation of comparison tasks using benchmarks.
  • Investigating the possibility of combining GPT’s ease of specification writing with Rosette’s correctness guarantees.

Example Task: Integer Bit Check

We tasked both GPT and Rosette with solving a synthesis problem: determining if an integer fits within a specified bit-width (e.g., 8 bits). GPT generated a program, and Rosette verified its correctness, returning an unsat result.

GPT Prompt: Write a Racket program that takes in an integer x and returns true if x can be represented with 8 bits.

GPT Output: (define (intI-gm_?? x) (and (< -129 x) (< x 128)))

Rosette's verification query confirmed the program's correctness.
